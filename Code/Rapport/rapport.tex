\subsubsection{Explication de la structure Note}
La structure \texttt{Note} reçoit et stocke les resultats academiques d'un etudiant
pour chaque matière de la filiere, ainsi que le coefficient correspondant. 
Par exemple, pour un étudiant donné, cette structure va enregistrer ses notes au fur et à mesure. 
Ensuite, grace au pointeur \texttt{struct Note *note\_suivante}, la structure se connecte à la note
suivante du même étudiant.C'est ce mécanisme qui forme une liste chaînée de notes propre à chaque etudiant, 
permettant de stocker un nombre dynamique de modules sans utiliser de tableau statique.

\lstinputlisting[linerange=12-21]{../structure.h}

\subsubsection{Explication de la structure Etudiant}
La structure \texttt{Etudiant} regroupe l'ensemble des informations d'un étudiant de la filière, telles que son nom, son 
prénom, son numéro de dossier (identifiant unique) et son état d'inscription (via un booleen).
Le pointeur \texttt{modules}  est essentielle : il pointe vers la première note de l'etudiant, liant ainsi l'etudiant 
à ses résultats.Concernant le pointeur \texttt{etudiant\_suivant} elle pointe sur l etudiant suivante creent ainsi 
une liste chaine

\section{Etude de Complexite }
Nous allont etudier la complexite de toute les fonctions dans le meillieur et pire des cas.
Soit \texttt{n} : le nombre d etudiant inscrite , et \texttt{m}: le nombre de module de chaque etudiant.





\subsubsection{Complexite du Fonctions inscrire\_etudiant}
Cette fonction ne contient que des operations elementaire .Donc par sommation des complexite de chaque ligne de code on obtient un O(1).





\subsubsection{Complexite du Fonctions note\_etudiant}
Dans cette fonction nous avons deux boucle de controle de saisie qui ne depand ni de \texttt{n} ni de \texttt{m}.
Mais du nombre d ereur de l utulisateur donc ce sont des O(1).
Ce pendant nous avons une troisieme boucle imbrique : \texttt{while (note\_restante->note\_suivante != NULL)} ,ce dernier parcour tout les note d un etudiant
jusqu a la fin pour y ajouter la nouvelle note donc le boucle tourne \texttt{m} fois 
La boucle principlale \texttt{while (nouveaux\_matiere == 2)} s execute aussi \texttt{m} fois car la fin du boucle 
depant du nombre de \texttt{matiere} saisie par l utulisateur 
De ce fait nous avons :
Dans le meillieur des cas O(1).
Dans le pire des cas O($m^2$).







\subsubsection{Complexite du Fonctions liste\_de\_la\_classe}
Cette fonction d affichage ne comporte qu une seule boucle \texttt{while (actuel\_liste != NULL)} . Cette boucle parcour toute les etudiants enregistre dans la filiere donc 
elle tourne \texttt{n} fois , donc nous aurons :
Dans le meillieur des cas : O(1).
Dans le pire des cas : O(n).








\subsubsection{Complexite du Fonctions bulletin}
Dans cette fonction nous avons une boucle imbrique : \texttt{while (actuel\_liste\_bulletin != NULL)}, cette boucle parcour l ensemble des module de chaque etudiant,(\texttt{m} fois).
Une boucle principlale : \texttt{while (actuel\_liste\_info != NULL)} qui parcour l ensemble des etudiant (\texttt{n} fois).
Ainsi nous avons :
Dans le meillieur des cas : O(1).
Dans le pire des cas : O(nxm).







\subsubsection{Complexite du Fonctions recherche\_et\_modification\_etudiant}
La fonction \texttt{recherche\_et\_modification\_etudiant} comporte deux boucle imbrique :
\texttt{while (etudiant\_actuel != NULL \&\& etudiant\_actuel->numero\_dossier != id\_rechercher)} qui s execute tantque l etudiant recherche n est pas trouver et la liste n est pas fini.
\texttt{ while (actuel->note\_suivante != NULL)} qui parcour toute les note d un etudiant.
Et une boucle principlale \texttt{ while (votre\_choix == 3)} qui ne depand pas du nombre d etudiant mais de l utulisateur donc c est un O(1).
Donc on a :
Dans le meillieur des cas : O(1).
Dans le pire des cas : O(n) + O(m).








\subsubsection{Complexite du Fonctions suprimer\_etudiant}
Dans cette fonction on a une boucle principlale \texttt{ while (votre\_choix == 4)} qui ne depand pas du nombre d etudiant mais de l utulisateur donc c est un O(1).
Et une boucle interne \texttt{while (actuel\_etudiant != NULL \&\& id\_supprimer != actuel\_etudiant->numero\_dossier)} qui parcour les etudiants en comparant leur id.
Donc on a :
Dans le meillieur des cas : O(1).
Dans le pire des cas : O(n).









\subsubsection{Complexite du Fonctions maximum}
